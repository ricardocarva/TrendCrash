The subquery ‚Äúavailable_traffic_details‚Äù (ATD) lists all of the various traffic details, such as a speedbump, roundabout or railroad crossing, that is stored in each state for a given time frame.

This subquery is joined with the accident, trafficdetails, locationdetails and state tables to ensure the necessary info is available. Then all of this is wrapped up in a subquery titled ‚ÄúIQ‚Äù.

The purpose of this is to join IQ with the driverpopulation table so that we could determine the accident count divided by the driver population to determine the accident rate. About The Query

This query aims to determine the effect on Covid19 and traffic accidents. Our first inner subquery ‚Äúbeforelockdown‚Äù determines the number of accidents in 2019 which is the year before covid.

It is wrapped into the subquery ‚Äúacc_covid.‚Äù Acc_covid gathers all of the accident data from various tables by day. This is then merged with driverpopulation to get the total number of drivers to get our output.

The main variables to compare are the accident rate(in 2020) and the difference in trips between 2019 and 2020 (trips in 2019 minus the trips in 2020). As the difference in trips goes down, we would expect the accident rate to go up because that means there were more people driving in 2020 at that time.

Value To The Customer

By analyzing the impact of COVID-19 on traffic accidents, our software enables users to identify patterns in mobility and safety during unusual events, aiding in more effective emergency response and policy-making during crises.

In this query we examine the effect of unemployment rate and accident rates. We create two primary subqueries, acc_unemp and qualrows.

Both subqueries join the appropriate tables and find the state, month, year, unemployment rate and accident count. What differs is that qualrows also has the dynamic queries so it has only the rows that meets our criteria.
